# قالب ملفات الأوامر — `cmds/*.js`

هذا الملف هو **المرجع الرسمي الوحيد** لطريقة كتابة أي أمر جديد في هذا المجلد.
اتّبعه بحذافيره وستحصل تلقائياً على:

- تحميل الأمر عند إقلاع البوت (وعند `.up` لاحقاً، بلا إعادة تشغيل).
- ظهوره مصنّفاً بشكل صحيح في `help`.
- ظهور وصفه وكل طرق استخدامه وبدائله وصلاحيته وكولداونه في `help <اسم_الأمر>`.

> **القاعدة الذهبية:** `help.js` لا يعرف شيئاً عن أي أمر بعينه. كل معلومة تظهر
> في `help` مصدرها الوحيد هو `config` الخاص بملف الأمر نفسه. إن أردت تغيير
> وصف أمر، أو إضافة طريقة استخدام جديدة له، أو إخفاءه، أو تغيير تصنيفه —
> عدّل ملف الأمر نفسه فقط. لا تحتاج (ولا يجب) أن تلمس `help.js` أبداً لهذا
> الغرض.

---

## 1. القالب الكامل

```js
// cmds/example.js
"use strict";

module.exports = {
  config: {
    // ── إلزامي ──────────────────────────────────────────────
    name: "example",              // اسم الأمر (بأحرف صغيرة، بدون مسافات، فريد)

    // ── إلزامي فعلياً (بدونه: يُصنَّف "غير مصنف" ويظهر في
    //    قسم "أوامر أخرى" أسفل قائمة help) ─────────────────────
    category: "أدوات",            // القسم الذي يظهر تحته الأمر في help

    // ── إلزامي فعلياً (بدونه: "لا يوجد وصف" في help) ──────────
    description: "وصف قصير من سطر واحد لما يفعله الأمر",

    // ── اختياري لكن موصى به بشدة ────────────────────────────
    aliases: ["مثال", "ex"],      // أسماء بديلة لاستدعاء نفس الأمر

    // كل "طرق الاستخدام" الممكنة للأمر — واحدة لكل صيغة/سلوك مختلف.
    // كل سطر يظهر كما هو في help <اسم_الأمر>. استخدم {pn} بدل رمز
    // البادئة، وضع اسم الأمر مباشرة بعده (بلا مسافة) — راجع القسم 3.
    usage: [
      "{pn}example <نص> — يكرر النص",
      "{pn}example رد على صورة — يصف الصورة",
    ],

    role: 0,        // 0 الجميع | 1 مشرف | 2 مراقب | 3 مميز(VIP) | 4 مطوّر — افتراضي 0
    countDown: 5,   // كولداون بالثواني بين استخدامين لنفس المستخدم — افتراضي 3

    hidden: false,  // true لإخفاء الأمر عن قائمة help (يبقى يعمل عادياً)
  },

  // ── إلزامي: نقطة الدخول ─────────────────────────────────────
  // اسم إحدى هذه الدوال بالضبط: onStart (الأكثر استخداماً) أو run
  // أو execute — لأوامر بلا بادئة/شرطية (مثل sing.js) استخدم onChat.
  onStart: async ({ api, event, args, message }) => {
    const { threadID, messageID } = event;
    // args = كل الكلمات بعد اسم الأمر
    await message.reply("مرحباً! الأمر شغال ✅");
  },
};
```

## 2. الحقول بالتفصيل

| الحقل | إلزامي؟ | التأثير |
|---|---|---|
| `config.name` | **نعم** | بدونه يتجاهل المحمِّل الملف بالكامل — لا يُسجَّل كأمر إطلاقاً. |
| `onStart` / `run` / `execute` / `onChat` | **نعم (واحدة منها على الأقل)** | بدونها الملف لا يُعتبر أمراً قابلاً للتنفيذ. |
| `config.category` | مستحسن | يحدد القسم في القائمة الرئيسية لـ `help`. غيابه → "غير مصنف" + تحذير في الـ console عند التشغيل. |
| `config.description` | مستحسن | السطر الذي يظهر بجانب اسم الأمر في القائمة الرئيسية وفي `help <أمر>`. |
| `config.usage` | مستحسن بشدة | مصفوفة نصوص، كل عنصر = طريقة استخدام كاملة مستقلة. هذا هو الحقل الذي يُظهر "كل طرق الاستخدام" في `help <أمر>` (راجع القسم 3). |
| `config.aliases` | اختياري | تظهر تلقائياً في `help <أمر>` تحت "🔗 البدائل"، وتُسجَّل كطرق استدعاء فعلية للأمر. |
| `config.role` | اختياري (افتراضي `0`) | يُطبَّق فعلياً في `index.js` لمنع من دون الصلاحية، ويظهر في `help <أمر>`. |
| `config.countDown` | اختياري (افتراضي `3`) | يُطبَّق فعلياً كمهلة تبريد، ويظهر في `help <أمر>`. |
| `config.hidden` | اختياري (افتراضي `false`) | إن كان `true` لا يظهر الأمر في `help` أو `help all` إطلاقاً، لكنه يبقى يعمل بشكل طبيعي عند استدعائه مباشرة (مناسب لأوامر صيانة مثل `up`، أو أوامر تُفعَّل تلقائياً بدون كتابة الاسم مثل `fb`). |

## 3. صيغة `usage` — كل طرق الاستخدام

اجعل كل عنصر في المصفوفة **سطراً كاملاً مستقلاً بذاته**، بصيغة:

```
"{pn}اسم_الأمر [وسيط] [وسيط اختياري] — شرح مختصر لما تفعله هذه الصيغة تحديداً"
```

- **`{pn}`** يُستبدل تلقائياً بعلامة البادئة الموحّدة في التوثيق (`.`). ضع
  اسم الأمر **مباشرة بعده بلا مسافة**: `"{pn}yt <اسم>"` وليس `"{pn} yt <اسم>"`
  (الفرق بينهما هو الفرق بين `.yt <اسم>` الصحيحة و `. <اسم>` الخاطئة).
- أدرج **كل** صيغة استخدام حقيقية للأمر كسطر منفصل — إن كان الأمر يدعم
  الرد على رسالة/صورة/فيديو كطريقة استخدام بديلة، وثّق ذلك أيضاً كسطر مستقل
  (مثال: `"رد على فيديو + {pn}sub <نص> — يضيف ترجمة على الفيديو"`).
- إن لم توفّر `usage` إطلاقاً، سيبقى الأمر يعمل ويظهر في `help` بوصفه فقط،
  لكن `help <أمر>` لن يعرض قسم "طريقة الاستخدام".

**مثال حقيقي (مطابق لأمر `sub`):**

```js
usage: [
  "رد على فيديو + {pn}sub <النص> — ترجمة ثابتة طوال الفيديو",
  "رد على فيديو + {pn}sub\\n00:01 - 00:03 | نص\\n00:04 - 00:07 | نص آخر — ترجمة زمنية (SRT)",
],
```

## 4. التصنيف (`category`) — كيف يُعرض في `help`

قيمة `category` نص حر بالكامل من اختيارك، لكن `help.js` يجمع الأوامر تحت
عناوين أقسام موحّدة (`CATEGORY_SECTIONS` داخل `cmds/help.js`) بمطابقة قيمة
`category` — أو اسم الأمر نفسه — مع إحدى القوائم المعرّفة هناك (تصنيفات مثل
"وسائط"/"media"، "أدوات"/"tools"، "إشراف"/"admin"، إلخ).

- إن استخدمت قيمة `category` تطابق (بغض النظر عن حالة الأحرف) واحداً من
  الأقسام الموجودة، سيُصنَّف أمرك معها تلقائياً — **لا حاجة لتعديل
  `help.js`**.
- إن أدخلت تصنيفاً جديداً كلياً غير موجود من قبل (مثلاً `"موسيقى"` بينما
  لا يوجد قسم بهذا الاسم)، سيظهر أمرك تحت "أوامر أخرى" في أسفل القائمة، مع
  تحذير في الـ console يوضّح ذلك. عندها فقط، أضف سطراً واحداً في
  `CATEGORY_SECTIONS` بأعلى `cmds/help.js` لربط هذا التصنيف بعنوان قسم
  (هذا التعديل الوحيد المسموح على `help.js`، وهو تعديل *عرض/تجميع* عام،
  وليس بيانات خاصة بأمر معيّن).

## 5. أمثلة سريعة (مفاهيم، وليست بالضرورة ملفات مُهيكَلة بالكامل بعد)

- **أمر عادي ببادئة (`onStart`)**: مثل `cmds/quran.js`, `cmds/tr.js`.
- **أمر بلا بادئة يُفعَّل عبر كشف نمط في النص (`onChat`)**: مثل `cmds/sing.js`
  (يُفعَّل بكتابة `sing <اسم>` مباشرة دون المرور عبر موجّه الأوامر
  المركزي؛ يحتاج مع ذلك نفس حقول `config` القياسية ليظهر بشكل صحيح في
  `help`).
- **أمر مخفي عمداً (`hidden: true`)**: مثل `cmds/fb.js` (يعمل تلقائياً عند
  اكتشاف رابط فيسبوك، ليس أمراً يُكتب يدوياً)، و`cmds/up.js` (أمر صيانة
  داخلي للمشرفين فقط).

> ملاحظة: بعض هذه الملفات لا تزال قيد التحديث للقالب الجديد بالكامل (راجع
> القسم 6) — الأهم هو الشكل العام (`onStart` مقابل `onChat`، ومتى يُستخدم
> `hidden`)، وليس أن تكون هذه الملفات مرجعاً كاملاً لكل حقل.

---

## 6. ⚠️ لا يوجد دعم للحقول القديمة

هذا القالب **إلزامي وليس اختيارياً**. `utils/commandMeta.js` يقرأ فقط:
`config.description` و`config.usage` (و`category`/`aliases`/`role`/
`countDown`/`hidden`). الحقول القديمة من إصدارات سابقة مثل
`shortDescription.ar`، `guide.ar`/`guide.en`، `guides`، `extra`، أو
`cooldown` (بدل `countDown`) **لم تعد مقروءة إطلاقاً**.

أي ملف أمر لم يُحدَّث بعد للقالب الجديد سيظهر في `help` بوصف تحذيري
("⚠️ لا يوجد وصف — لم يُهيكَل بعد") وبدون قسم "طريقة الاستخدام"، لكنه
سيبقى يعمل وظيفياً بشكل طبيعي عند استدعائه — فقط عرضه داخل `help` هو
الناقص إلى حين تحديث `config` الخاص به وفق هذا الملف.

**لا تُعِد أي حقل قديم مع حقل جديد مكافئ له في نفس الملف** (مثلاً لا تكتب
`cooldown` و`countDown` معاً، ولا `guide` و`usage` معاً) — إمّا القالب
الجديد بالكامل، أو لا شيء. الهدف تجنّب أي تهجين بين الهيكلين.
